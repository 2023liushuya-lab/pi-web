# pi-web 文件卡片功能

Agent 使用 `write` / `edit` 工具生成或修改文件后，聊天窗口中不再只显示一行枯燥的工具调用日志，而是以**内嵌卡片**形式直接展示文件内容。

---

## 卡片包含

- 📄 **文件名、大小、操作类型**（Created / Modified）——一眼看清生成了什么
- 📋 **内容预览**——前 18 行直接在对话中可见，不需要切到其他面板
- 🔽 **可折叠**——长文件可以收起，保持对话整洁
- 🔗 **Open**——一键在右侧 FileViewer 中完整查看
- ⬇️ **Download**——直接下载到本地

## 实现原理

### 数据流

```
Agent writes file
  ↓
ChatWindow (有 onOpenFile)
  ↓
MessageView → AssistantMessageView → BlockView
  ↓
ToolCallBlock
  ↓ (if toolName === "write" / "edit" && 执行成功)
FileCard
  ├── fetch /api/files/{path}?type=read
  ├── 渲染文件预览
  └── 点击 Open → onOpenFile(path, name) → AppShell 打开 FileViewer
```

### 文件类型检测

- **图片**（png、jpg、gif、webp、svg 等）→ 缩略图预览
- **代码/文本** → 语法高亮预览（前 18 行）
- **Markdown** → 纯文本预览
- **其他** → 文件图标 + 下载按钮

## 改动清单

| 文件 | 操作 | 说明 |
|------|------|------|
| `components/FileCard.tsx` | **新建** | 文件卡片组件 |
| `components/MessageView.tsx` | 修改 | `ToolCallBlock` 检测 write/edit 成功后渲染卡片；逐层透传 `onOpenFile` |
| `components/ChatWindow.tsx` | 修改 | Props 新增 `onOpenFile`，传给两个 `MessageView` |
| `components/AppShell.tsx` | 修改 | 把已有的 `handleOpenFile` 传给 `ChatWindow` |

> 纯前端改动，不影响任何 API 或后端逻辑。Agent 发送消息、调用工具的流程完全不变——只是渲染层换了一种更友好的展示方式。
